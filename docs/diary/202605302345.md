# 2026-05-30 23:45 — Rollup → esbuild

## Quyết định
Chuyển build pipeline từ rollup (+ 3 plugins) sang esbuild. 4 devDeps → 1.

## Lý do
- Project không cần plugin ecosystem của rollup (không TS, không JSX, không CSS-in-JS, không custom transform). Mọi thứ trước đây cần plugin (`nodeResolve`, `json`, `terser`) đều là built-in trong esbuild.
- esbuild nhanh hơn rất nhiều — full build ~30ms vs rollup ~vài giây.
- Ít dep hơn = ít drift hơn.

## Thay đổi
- `package.json`: gỡ `rollup`, `@rollup/plugin-node-resolve`, `@rollup/plugin-json`, `@rollup/plugin-terser`; thêm `esbuild`.
- `build.js`: viết lại bằng `esbuild.build({...})`. API gần giống rollup, gọn hơn. Mọi behavior giữ nguyên (ESM server, ESM background, IIFE content/dashboard, manifest auto-gen từ hosts.js, copy CSS).
- `.github/copilot-instructions.md`, `docs/plugin-dev-guide.md`: cập nhật "rollup + terser" → "esbuild".

## Verify
- `npm run build` → green, output structure y hệt:
  - `build/server/index.js` 6.8kb
  - `build/browser/background/index.js` 10.7kb
  - `build/browser/facebook/content.js` 8.3kb
  - `build/browser/dashboard/index.js` 11.0kb
  - manifest auto-generated với `https://*.facebook.com/*`.
- Tổng thời gian build: ~30ms.

## Hệ quả
- 1 devDep duy nhất.
- Build script ngắn hơn (~100 dòng vs ~160).
- Không có lib bên ngoài cần stub/patch.
