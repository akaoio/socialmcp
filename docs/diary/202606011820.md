# 2026-06-01 18:20 — Debug MCP tools: screenshot, getdom, getaxstree, ocr

## What changed

Added 4 debug MCP tools that let an AI agent (or developer) inspect the current state of a social platform tab without any platform-specific code.

### New files
| File | Role |
|------|------|
| `src/browser/background/screenshot.js` | Activates the owned tab, calls `captureVisibleTab`, returns `{ dataUrl }` |
| `src/browser/background/getdom.js` | `executeScript` → `document.documentElement.outerHTML`, returns `{ html }` |
| `src/browser/background/getaxstree.js` | `executeScript` → compact ARIA tree walker (interactive + landmark elements, max depth 8, text truncated at 120 chars), returns `{ tree }` |
| `src/server/ocr.js` | Server-side OCR: strips data-URL prefix → saves temp PNG → `execFile('tesseract', ...)` → reads output txt → cleanup |

### Modified files
- **`src/browser/background/dispatch.js`** — added `BUILTINS = { screenshot, getdom, getaxstree }` checked before `plugin.background` lookup; these actions are platform-agnostic and need no plugin handler
- **`src/browser/manifest.json`** — `host_permissions` changed from `https://*.facebook.com/*` to `<all_urls>` (required for `captureVisibleTab` on any tab)
- **`src/server/index.js`** — 4 new tools registered:
  - `screenshot` → bridge `screenshot` → returns MCP `image` content type (base64 PNG)
  - `getdom` → bridge `getdom` → JSON `{ html }` wrapped in text reply
  - `getaxstree` → bridge `getaxstree` → JSON `{ tree }` wrapped in text reply
  - `ocr` → bridge `screenshot` + server-side `ocr()` → JSON `{ text }`
- **`install.sh`** — `tesseract-ocr` added to `--server` apt packages

## Why these tools

Tests and agents need a feedback loop. Without visibility into what the browser is actually showing, it's impossible to verify that a `post` or `scan` action did the right thing. These tools give agents (and developers) three complementary views:

- **screenshot** — visual: catch layout issues, verify UI state
- **getdom** — structural: precise element targeting, CSS selector debugging
- **getaxstree** — semantic: ARIA/role-based view, useful for writing resilient selectors
- **ocr** — text: extract visible text when the DOM obscures it (canvas, SVG, images)

## Tradeoffs

- **`<all_urls>` in host_permissions** — broader than strictly needed for Facebook, but debug tools are inherently cross-platform. The extension is developer-only, not published to the store.
- **ARIA tree walker is in-page JS** — injected by `executeScript`; output is a formatted string. A structured JSON tree would be richer but the string format is more LLM-friendly and keeps the diff small.
- **OCR is server-side** — Tesseract can't run in a browser extension. The tool re-uses the `screenshot` action rather than adding a new bridge endpoint.
- **Tesseract optional** — `ocr.js` throws a friendly error if `tesseract` binary isn't found; the other 3 tools work without it.
