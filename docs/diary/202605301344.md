# 2026-05-30 13:44 — Multi-image upload across all platforms

## What changed

Implemented bulk DataTransfer multi-image upload for all 4 platforms, inspired by MultiPost-Extension's technique.

### facebook/post.js (already done previous session)
Full rewrite with `setfiles()` bulk DataTransfer helper, media-first order (attach before typing to avoid album-mode text loss), `findtrigger()` walking DOM from Photo/video anchor, re-fetch dialog after attach.

### facebook/postpage.js
- Added `setfiles()` helper (same bulk DataTransfer pattern).
- Changed signature from `{ content, image }` to `{ content, media = [], image }`.
- `files = media?.length ? media : (image ? [image] : [])` — backwards compatible with old single-image callers.
- Media-first: click Photo/video → setfiles → sleep(4000) → re-fetch dialog → type text.
- Removed old `attachphoto()` function (sequential, single-file, waited for Remove Post Attachment ack).

### x/post.js
- Added `setfiles()` with max-4 cap (X platform limit).
- Signature: `{ content, media = [] }`.
- File input: `input[data-testid="fileInput"]` with `input[type="file"]` fallback.
- Media attached after typing, sleep(3000) for upload.

### instagram/post.js
- Full rewrite. Old code used broken `:has-text("Share")` Playwright selector.
- `setfiles()` — no native setter needed for Instagram (not React-managed input).
- `clicknext()` — loops `div[role="button"][tabindex="0"]` by textContent "Next".
- `clickshare()` — same pattern for "Share".
- Flow with media: setfiles → Next (crop) → Next (filter) → caption → Share.
- Flow without media: skip to caption directly → Share.
- Caption fallback: tries `aria-label="Write a caption…"` then `aria-placeholder` contains "caption".

### threads/post.js
- Full rewrite. Old code used broken `:has-text("Post")` Playwright selector.
- Added `setfiles()`, `clickpost()` (loops `[role="button"]` by textContent "Post").
- Signature: `{ content, media = [] }`.
- Flow: type content → if media: find mediainput → setfiles → sleep(3000) → clickpost.

### threads/selectors.js
- Added `mediainput: 'input[type="file"]'`.

## Tradeoffs / notes

- `setfiles()` is duplicated in each platform file (not shared via `common/utils.js`) — content scripts are isolated IIFE bundles; sharing would require a common bundle or copy-paste. Copy-paste keeps it simple.
- Instagram's carousel upload (multiple files in single DataTransfer) relies on Instagram accepting all files at once on the `composerimg` input — matches MultiPost's approach.
- Threads `mediainput` is a broad `input[type="file"]` — only one such input exists in the compose modal so this is fine.
- X max-4 cap is enforced in `setfiles()` via `.slice(0, 4)`.

## Build
`npm run build:ext` — clean exit, all 5 content bundles emitted.
