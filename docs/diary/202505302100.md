# 202505302100 — Refactor content scripts into per-action files

## Thay đổi

Tách từng file `content.js` monolithic (mỗi platform ~300 dòng) thành cấu trúc module hóa:

```
src/browser/
  common/
    utils.js      — wait, sleep, type, press (dùng chung 4 platform)
    scroll.js     — scroll(params, { article, text, author, link })
  <platform>/
    selectors.js  — const S = { ... }
    post.js
    comment.js
    react.js
    scroll.js     — gọi common/scroll với selectors của platform
    search.js
    follow.js
    unfollow.js
    message.js
    profile.js
    content.js    — entry point: import tất cả, khai báo HANDLERS, router
```

## Lý do

User yêu cầu "mỗi 1 hành động cần 1 file riêng, không file nào drift file nào". Cấu trúc cũ gộp 9 handlers + selectors + utilities vào 1 file → khó bảo trì khi platform thay đổi selector.

## Quyết định thiết kế

- **common/utils.js**: Trước đây `wait`, `sleep`, `type`, `press` bị duplicate 4 lần. Rollup tree-shakes → mỗi bundle chỉ include những gì dùng.
- **common/scroll.js**: Thuật toán scroll giống nhau ở cả 4 platform, chỉ khác selector key name (`tweettext` vs `postcontent`). Dùng signature `scroll(params, { article, text, author, link })` để normalize.
- **content.js vẫn là entry point**: build.js dùng `src/browser/<p>/content.js` → không cần sửa build config.
- **selectors.js per platform**: Tất cả CSS selector tập trung 1 chỗ, import vào từng action file → khi Facebook/X thay markup chỉ sửa 1 file.
