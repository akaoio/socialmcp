# 2025-06-02 — Naming convention drift fixes

## What changed

Full audit and fix of all naming convention violations across `src/`.

### Server

- **`mcp.js` → 3 files**: `schema.js` (`schema`), `mcpserver.js` (`mcpserver`), `stdioservertransport.js` (`stdioservertransport`). Each file = one exported name matching filename.
- **`bridge.js`**: converted from a `Bridge` class to a singleton object. Eliminates the naming conflict where both the import and the instance were named `bridge`. Internal functions renamed: `toDataUrl`→`todataurl`, `resolveMedia`→`resolvemedia`, `lastPeerAt`→`lastpeerat`, `peerStale`→`peerstale`.
- **`index.js`**: switched to `import { bridge } from './bridge.js'`, removed `const bridge = new bridge().start()` (singleton, no instantiation), removed dead `mediatourl` function + unused `readFileSync`/`extname` imports.
- **`schema.js`**: `class Sch` → `class sch`.
- **`ocr.js`**: switched from system `tesseract` CLI to `tesseract.js` npm package. No system dependency. `cachePath` set to `/tmp/socialmcp-tessdata` so `eng.traineddata` never lands in the repo. Parameter renamed `dataUrl`→`dataurl`.

### Browser

- **`panel.js`**: export `html` → `panel`; `mount.js` updated to match.
- **`screenshot.js`**: returns `{ dataurl }` (was `{ dataUrl }`).
- **`getaxstree.js`**: local `isLeaf` → `isleaf`.
- **`grouptab.js`**: variable `groupId` → `groupid`; Chrome API call uses explicit `{ groupId: groupid }` to preserve the required key name.

### Tests

- `tests/debug.spec.js`: all 4 tests run unconditionally (OCR now always available via npm); `dataUrl` → `dataurl`.
- `tests/post.spec.js`: `toDataUrl` → `todataurl`.

## Why

Hard rule: all function names, file names, and exported identifiers must be single lowercase words `[a-z]` only. These were accumulated drift violations found in a deep automated scan.

## Tradeoffs

- `bridge.js` as a singleton is slightly less testable in isolation but the previous `Bridge` class pattern created a real naming conflict and `start()` was already a no-op (server started at module load). Singleton is cleaner here.
- `mcp.js` split into 3 files increases file count but keeps each file aligned with the one-export-per-file rule.
