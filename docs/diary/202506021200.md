# 2025-06-02 — Dashboard removal, structural refactor, docs overhaul

## Tóm tắt

Session lớn tập trung vào dọn dẹp kiến trúc triệt để, khai tử dashboard UI, và chuẩn hoá convention "feature = folder, function = file".

## Thay đổi chính

### 1. Feature = folder, function = file (sửa vi phạm)

Phát hiện 3 file vi phạm (nhiều hơn 1 exported function trong 1 file):

- `src/server/bridge.js` → chuyển thành `src/server/bridge/` (bridge.js + todataurl.js + resolvemedia.js)
- `src/browser/background/findtab.js` → `src/browser/background/findtab/` (findtab.js + gettabs.js)
- `src/browser/dashboard/init.js` → folder (bị xoá khi dashboard bị khai tử)

### 2. Khai tử dashboard UI

Toàn bộ `src/browser/dashboard/` và `src/browser/platform/facebook/dashboard/` bị xoá.  
Dashboard đã là overhead không cần thiết sau khi có relay page và test automation.

**Thay thế:** `src/browser/relay/relay.html` + `relay.js` — trang extension tối giản chỉ expose `window.dispatch()` cho tests. 5 dòng code, không có UI.

Cũng xoá: `src/browser/background/opendashboard.js`, `chrome.action.onClicked` listener.

### 3. Builtin actions → `src/browser/builtin/`

Các action platform-agnostic (`screenshot`, `getdom`, `getaxstree`) trước đây nằm flat trong `src/browser/background/` vi phạm "feature = folder".

Chuyển sang:
```
src/browser/builtin/
  screenshot/screenshot.js
  getdom/getdom.js
  getaxstree/getaxstree.js
```

### 4. Dead code dọn sạch

- `src/browser/common/filetourl.js` — không ai import, xoá.
- Plugin manifest `plugin.js` của facebook: xoá `css`, `dashboard`, `mount` fields.

## Lý do

Dashboard tạo ra nhiều coupling giữa core và plugin mà không cần thiết. Relay page nhỏ hơn, đơn giản hơn, và đủ cho test automation. Convention "feature = folder" quan trọng vì nó ngăn file lớn lên và giữ code easy to navigate theo feature.

## Tradeoff

Mất khả năng manual test qua UI — nhưng test automation đã đủ mạnh để thay thế. Ai muốn test manual có thể dùng MCP inspector.
