# noVNC setup + real E2E test suite

## Thay đổi

### 1. noVNC stack trên Orange Pi 5

Mục đích: login Facebook thật từ điện thoại (SSH headless, không có màn hình).

Stack: Xvfb :1 → openbox → x11vnc → websockify (port 6080) → noVNC web UI tại `http://pi.akao.io:6080/vnc.html`.

Vấn đề gặp phải:
- Playwright's bundled Chromium crash liên tục trên ARM (GPU/zygote subprocess crash)
- Dùng system Chromium `/usr/lib/chromium/chromium` với `--no-sandbox --no-zygote --single-process`
- x11vnc crash do xdamage → restart với `-noxdamage`
- Thêm watchdog script tự restart Chromium khi crash
- VNC password tại `/home/x/.vncpasswd` (ngoài repo)

### 2. Extract Facebook cookies

Sau khi login, extract cookies từ Chromium SQLite DB (`~/.config/chromium/Default/Cookies`):
- Encrypted với AES-128-CBC, key = PBKDF2("peanuts", "saltysalt", 1, 16), IV = b' '*16
- Format: b"v10" + 32-byte prefix (không dùng) + actual value
- Giải mã bằng `pycryptodome`; plaintext bắt đầu từ byte 32 của decrypted output
- Lưu tại `/tmp/fb_cookies.json` (ngoài repo)

## Tradeoffs

- `parser.spec.js` (mock HTML regression test cho getpages selector) đã bị xóa vì user không muốn mock. Nếu FB đổi DOM, `facebook.spec.js` scan test sẽ catch.
- Cookies hết hạn sau vài tuần — cần login lại qua noVNC.
- `--single-process` Chromium kém ổn định hơn multi-process nhưng là cách duy nhất tránh subprocess crash trên ARM Armbian.
