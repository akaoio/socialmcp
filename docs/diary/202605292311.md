# 2026-05-29 23:11 — postpage 2-step navigation fix

## Problem

Posting to a Facebook Page failed because the "Switch Now" banner uses
language-specific text (`aria-label="Switch Now"` / "Chuyển Ngay" / etc.).
The earlier fix tried navigating directly to `latest/home?asset_id={page_id}`,
but that admin URL's "Create Post" button opens Meta Business Suite — not an
inline post composer.

## Debug Findings

Running `fb-debug-adminpost.mjs` on `latest/home?asset_id=843538588847109`:
- "Create Post" button has no `aria-haspopup`, no `data-testid`
- Clicking it navigates to Meta Business Suite — no inline composer available

Running `fb-debug-switchbanner.mjs` on `/akaoofficial`:
- Composer (`[aria-haspopup]`) was **already present** — session was in page
  identity from a prior admin URL visit
- This revealed the key insight: visiting the admin URL switches the Facebook
  session to page identity, which persists for subsequent navigations

## Solution: 2-Step Navigation in background.js

For `postpage` action when `page_id` is available:

1. Navigate to `https://www.facebook.com/latest/home?asset_id={page_id}` →
   Facebook redirects to `business.facebook.com/…` and sets the session to
   act as the page identity (cookie-based)
2. Navigate to `https://www.facebook.com/{page_handle}` → page is now in
   page identity, composer shows "Share a thought…" with `aria-haspopup="menu"`

No language-specific selectors needed at all.

## Files Changed

### `src/browser/background.js`
Added 2-step navigation before the standard navigation logic:
```js
if (platform === 'facebook' && action === 'postpage' && params?.page_id && params?.page_url?.startsWith('http')) {
  const adminUrl = `https://www.facebook.com/latest/home?asset_id=${params.page_id}`;
  await navigate(tab.id, adminUrl);
  await navigate(tab.id, params.page_url);
  const updated = await chrome.tabs.get(tab.id);
  return sendmessage(updated.id, { action, params });
}
```

### `src/browser/dashboard.js`
`fbpost()` now passes `page_url: target` (the page handle URL) and
`page_id: page?.id` instead of substituting the admin URL as `page_url`:
```js
await dispatch('facebook', 'postpage', { page_url: target, page_id: page?.id, ...params });
```

### `src/browser/facebook/postpage.js` + `selectors.js`
No changes needed — `findtrigger` already uses structural selector
`[role="main"] [role="button"][aria-haspopup="menu"]` which matches the
"Share a thought…" trigger after identity switch.

## Verification

`fb-debug-postpage-e2e.mjs` confirmed:
- Step 1 admin URL redirects to `business.facebook.com` (sets page identity)
- Step 2 `/akaoofficial` finds composer via strategy s1 immediately
- Clicking trigger opens textbox — ready for posting

## Tradeoffs

- Two navigations per post (adds ~3-4 seconds to each postpage call)
- Requires the tab to be on a facebook.com URL initially (existing requirement)
- Requires `page_id` to be known; falls back to standard navigation if absent
