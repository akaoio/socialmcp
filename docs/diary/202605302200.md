# 2026-05-30 — Plugin architecture refactor

## What changed

Browser extension chuyển hoàn toàn sang kiến trúc plugin.

- Thêm `src/browser/plugins.js` làm registry trung tâm.
- `src/browser/{background,dashboard,common}/` được dọn sạch — không còn bất kỳ tham chiếu nào tới `facebook` hay platform cụ thể.
- Tạo `src/browser/platform/facebook/plugin.js` làm manifest cho FB plugin: `{ id, label, hosts, css, background, dashboard }`.
- FB dashboard panel chuyển vào `platform/facebook/dashboard/` (mount.js, panel.js, panel.css, state.js, log/post/scanpages/renderpages/setupimagepicker/loadfiles/clearpicker).
- FB background handler chuyển vào `platform/facebook/background/dispatch.js`.
- `background/findtab.js` nhận `hosts: string[]` thay vì hard-code platform.
- `background/dispatch.js` route theo registry: nếu `plugin.background[action]` có thì gọi, không thì fallback generic.
- `dashboard/init.js` rewrite thành loader: loop `plugins`, inject `<link>` CSS, build sidebar button, lazy-mount panel.
- `dashboard/index.html` và `dashboard/index.css` rút xuống còn shell thuần.
- Common utils gom về `src/browser/common/`: `sleep`, `wait`, `type`, `press`, `filetourl`.
- Bỏ `platform/facebook/selectors.js` gom chung; mỗi feature folder (`post/`, `scan/`, …) tự giữ `selectors.js` riêng — tránh bloat khi nhiều tính năng thêm vào.
- Per-plugin state đổi tên: `fbpages` → `pages`, `imageDataUrls` → `media`.
- `build.js` thêm bước `fs.cpSync('src/browser/platform', …, { filter: src.endsWith('.css') })` để mọi plugin CSS đều có mặt trong `build/browser/platform/**`.

## Why

User cực kỳ ghét drift. Trước đó FB code rò rỉ vào `background/` và `dashboard/`, khiến việc thêm platform mới đòi hỏi sửa code core. Sau refactor, thêm platform = tạo `platform/<id>/` + đăng ký vào `plugins.js` + entry trong `manifest.json` `content_scripts`. Không đụng tới `background/` hay `dashboard/`.

Hard rule mới: tên hàm và file luôn là một từ `[a-z]` viết thường. Filename khớp tên hàm export.

## Tradeoffs

- Mỗi plugin phải duplicate vài file (`dispatch.js`, `mount.js`, …). Đánh đổi này được chấp nhận vì plugin trở nên thực sự tự chứa, dễ copy-paste cho platform mới.
- Plugin CSS được load runtime qua `<link rel="stylesheet">` chứ không bundle. Đơn giản hoá build và cho phép plugin chỉnh style riêng mà không cần touch shell.
- Selectors per-feature làm số file tăng nhẹ, nhưng tránh được file `selectors.js` khổng lồ chứa hết mọi feature về sau.
