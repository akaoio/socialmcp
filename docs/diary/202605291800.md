## 1. Chuyển sang kiến trúc ZEN peer thực sự

### Vấn đề trước đó

`bridge.js` dùng `@akaoio/zen/lib/websocket.js` như một thư viện WebSocket thô — về bản chất vẫn là custom JSON protocol qua raw WebSocket:

```
server  →  ws.send(JSON.stringify({ id, platform, action, params }))
browser →  ws.send(JSON.stringify({ type: 'response', id, result }))
```

Không có gì là "ZEN" ở đây ngoài cái class `WebSocketServer`. Extension phải tự `register` platform, server phải tự quản lý connection map, pending map, seq counter phức tạp.

### Thay đổi: bridge.js → ZEN relay

```js
// Trước
import WS from '@akaoio/zen/lib/websocket.js';
const { Server: WebSocketServer } = WS;
this.wss = new WebSocketServer({ server });
this.connections = new Map();  // connId → { ws, platforms }
this.pending = new Map();      // msgId → { resolve, reject, timer }
conn.ws.send(JSON.stringify({ id, platform, action, params }));

// Sau
import ZEN from '@akaoio/zen/zen.js';
import '@akaoio/zen/lib/wire.js';
this.zen = new ZEN({ web: srv, file: false, axe: false });
zen.get('socialmcp').get('cmd').get(id).put(JSON.stringify({ platform, action, params, ts }));
```

`send()` bây giờ:
1. Ghi command vào `socialmcp/cmd/<id>` dưới dạng JSON string
2. Lắng nghe `socialmcp/res/<id>` bằng `.on()`
3. Sau khi nhận response: `.off()`, xóa cả cmd và res node (`.put(null)`), resolve/reject

Không còn connection map, không còn platform registration, không còn `ws.send()`.

### Thay đổi: background.js → ZEN peer client

```js
// Trước
socket = new WebSocket(WS_URL);
socket.addEventListener('open', () => {
  socket.send(JSON.stringify({ type: 'register', platforms: [...] }));
});
socket.addEventListener('message', async event => {
  const msg = JSON.parse(event.data);
  const result = await dispatch(msg.platform, msg.action, msg.params);
  socket.send(JSON.stringify({ type: 'response', id: msg.id, result }));
});

// Sau
zen = new ZEN({ peers: [ZEN_URL], axe: false });
zen.get('socialmcp').get('cmd').map().on(async (raw, id) => {
  if (!raw || typeof raw !== 'string') return;
  const cmd = JSON.parse(raw);
  if (Date.now() - (cmd.ts || 0) > 60000) return; // bỏ lệnh cũ
  zen.get('socialmcp').get('cmd').get(id).put(null); // xóa để không xử lý lại
  const result = await dispatch(cmd.platform, cmd.action, cmd.params);
  zen.get('socialmcp').get('res').get(id).put(JSON.stringify({ ok: result }));
});
```

Không còn `register`, không còn `backoff`, không còn `reconnect()`. ZEN tự reconnect.

### Import: dùng zen.js trực tiếp thay vì lib/server.js

`lib/server.js` kéo theo toàn bộ chuỗi: `lib/store.js` → `lib/rfs.js` → `lib/stats.js` → `lib/multicast.js` → `lib/dht.js` → `lib/rs3.js` → nhiều module Node.js nặng không cần thiết. Import trực tiếp `zen.js` + `lib/wire.js` (side-effect import để có opt.web WebSocket server support) là đủ.

## 2. Port 3456 → 8420

`3456` là port tự đặt tùy tiện từ session trước. `8420` là port mặc định chính thức của ZEN mainnet, định nghĩa trong `lib/xdg.js`:

```js
export const DEFAULT_PORT = (function () {
  const n = net();
  if (n === "mainnet") return 8420;
  if (n === "testnet") return 8421;
  ...
})();
```

Thay đổi ở: `src/server/bridge.js`, `src/browser/background.js`, comment trong `src/server/index.js`, `copilot-instructions.md`.

## 3. Xóa tàn dư zod: z → schema

### Vấn đề

`mcp.js` export `const z = { string, number, enum, array }` — tên `z` là convention của Zod, không có nghĩa gì với người đọc code lần đầu mà không biết context:

```js
import { McpServer, StdioServerTransport, z } from './mcp.js';
const platform = z.enum([...]);
content: z.string().describe('...')
```

Đây là technical debt: code trông giống Zod nhưng không phải, gây nhầm lẫn.

### Thay đổi

`mcp.js`: đổi `export const z` → `export const schema`

`index.js`: đổi toàn bộ import và usage:
```js
// Trước
import { ..., z } from './mcp.js';
const platform = z.enum(['facebook', 'x', 'instagram', 'threads']);
content: z.string().describe('Text content of the post'),

// Sau
import { ..., schema } from './mcp.js';
const platform = schema.enum(['facebook', 'x', 'instagram', 'threads']);
content: schema.string().describe('Text content of the post'),
```

Tổng cộng 9 chỗ thay thế trong `index.js`.

## 4. Build fixes

`zen.js` chứa `import("./service.js")` động — khi rollup bundle, nó không tìm thấy `./xdg.js` mà `service.js` import (file này không tồn tại ở root của package zen).

Giải pháp: thêm plugin `zenServiceStub` vào `build.js`:

```js
const zenServiceStub = {
  name: 'zen-service-stub',
  resolveId(id, importer) {
    if (id === './service.js' && importer?.replace(/\\/g, '/').includes('@akaoio/zen')) {
      return '\0zen-service-stub';
    }
  },
  load(id) {
    if (id === '\0zen-service-stub') return 'export default function(){}';
  },
};
```

Thêm `inlineDynamicImports: true` vào output của server và background để rollup không tách dynamic import thành chunk riêng.

Thêm `nodeResolve({ browser: true })` cho background bundle vì `@akaoio/zen/zen.js` cần resolve trong môi trường browser.

## Trạng thái cuối session

- `node build.js` → 7/7 outputs ✓, không warning
- Không còn `z.` trong codebase server
- Không còn raw WebSocket protocol trong bridge/background
- Không còn port 3456 ở bất kỳ đâu

## Tradeoffs

- `.on()` listener trong background.js nhận mọi update của bất kỳ `cmd/<id>` node nào, kể cả lần ta `.put(null)` để xóa — guard `if (!raw) return` xử lý case này.
- `file: false` trên relay: command không persist xuống disk. Server restart giữa chừng → client timeout, acceptable.
- `axe: false` trên cả hai đầu: tắt AXE clustering (không cần cho use-case local), tránh load `lib/axe.js` và peer discovery không cần thiết.
