# Integration test suite với Playwright

## Thay đổi

Thêm test suite tích hợp thực sự dùng Playwright để load extension Chrome và chứng minh hệ thống hoạt động end-to-end.

### Files mới
- `playwright.config.js` — cấu hình Playwright (workers: 1, timeout 30s)
- `tests/extension.spec.js` — Tier 1 + Tier 2 tests (luôn chạy)
- `tests/facebook.spec.js` — Tier 3, chỉ chạy khi có `FACEBOOK_COOKIES` env var
- `package.json` — thêm `"test"` script + `@playwright/test` devDep

### 3 loại test đang chạy

**Tier 1 — Extension structure + message chain (không cần mạng)**
1. `service worker starts` — background SW khởi động đúng URL
2. `dashboard renders facebook plugin` — dashboard mở, button "Facebook" visible
3. `message chain: unknown platform returns structured error` — gửi `platform: 'nope'` → nhận `{error: 'Unknown platform: nope'}` → chứng minh toàn bộ dashboard→SW→dispatch pipeline hoạt động

**Tier 2 — Content script injection + DOM parsing (localhost mock)**
4. `content script getpages parses mock facebook dom` — inject `facebook/content.js` vào trang HTML giả có DOM giống Facebook, gọi `getpages`, kiểm tra trả về đúng 2 pages từ mock

**Tier 3 — Real Facebook (skipped, cần `FACEBOOK_COOKIES`)**
- Chỉ scan, không post — tránh side effects

## Kỹ thuật đáng chú ý

### `--headless=new` + extension
Chrome extensions không chạy được trong headless chuẩn. Dùng trick:
```js
chromium.launchPersistentContext(udir, {
  headless: false,         // ngăn Playwright inject --headless flag
  args: ['--headless=new', '--load-extension=...'],
});
```
Kết quả: extension load + service worker start mà không cần display/Xvfb.

### Fresh tmp dir per run
Mỗi lần chạy tạo dir mới trong `/tmp/socialmcp-*` → tránh state bleed từ runs trước.

### Tại sao KHÔNG dùng `scan` làm smoke test
`findtab()` tự tạo tab Facebook nếu chưa có → smoke test không còn hermetic. Thay bằng invalid platform → response xác định.

### Content script test qua extension page
`chrome.scripting.executeScript` và `chrome.tabs.sendMessage` chỉ available từ extension context. Mở dashboard page, gọi từ đó thay vì từ SW `evaluate()` — ít moving parts hơn.

### Mock HTML vs. selector mismatch
`getpages()` dùng `[role="main"] [role="listitem"] a[href] span[dir]` và `filter()` parse tên + URL. Mock HTML được thiết kế match chính xác selectors đó. `id` = null là bình thường (cần `asset_id` trong href, không có trong mock).
